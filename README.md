# TEST PLAN — E-commerce QA Testing

## 1. Objetivo

El objetivo de este plan es validar el correcto funcionamiento de las funcionalidades principales de un sistema e-commerce, asegurando que los usuarios puedan registrarse, iniciar sesión, gestionar un carrito de compras y completar pagos de manera correcta.

---

## 2. Alcance

Se probarán las siguientes funcionalidades:

- Registro de usuario
- Inicio de sesión (login)
- Navegación de productos
- Gestión del carrito de compras
- Proceso de pago

---

## 3. Fuera de alcance

- Pruebas de seguridad avanzadas (penetration testing)
- Pruebas de rendimiento profundas (performance testing)
- Integraciones externas reales (pasarelas de pago reales)

---

## 4. Tipos de pruebas

- Pruebas funcionales
- Pruebas de validación de datos
- Pruebas de flujo (end-to-end)
- Pruebas negativas (inputs inválidos)

---

## 5. Entorno de pruebas

- Navegador: Google Chrome
- Sistema operativo: Windows
- Herramientas: 
  - Navegador web
  - Postman (para API)
  - GitHub (documentación)

---

## 6. Criterios de entrada

- La aplicación debe estar disponible
- Acceso a internet estable
- Usuario de prueba creado (si aplica)

---

## 7. Criterios de salida

- Todos los casos de prueba ejecutados
- Bugs críticos documentados
- Resultados registrados

---

## 8. Roles y responsabilidades

- QA Tester:
  - Diseño de casos de prueba
  - Ejecución de pruebas
  - Reporte de bugs

---

## 9. Riesgos

- Fallos en la conexión a internet
- Datos inconsistentes en la aplicación
- Limitaciones del entorno de pruebas

---

## 10. Entregables

- Test Plan
- Casos de prueba
- Reporte de bugs
Add initial api testing
